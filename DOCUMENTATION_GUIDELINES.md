# Project Documentation Guidelines

## The "Always Document" Rule
**Status: MANDATORY**

As per the project requirements established during the Shipping Page development:

1.  **Continuous Documentation**: Every section or major feature implemented MUST be explicitly documented in the relevant architectural files (e.g., `SHIPPING_PAGE_SECTIONS.md`).
2.  **Synchronized Updates**: Whenever a change is made to a section's logic, design, or technical specification, the documentation must be updated immediately to reflect the current "source of truth."
3.  **Technical Detail**: Documentation must include:
    *   Description of the feature.
    *   Technical specifications (CSS layout, classes, etc.).
    *   Animation logic (Keyframes, timing cycles, triggers).
    *   Responsive behavior.
4.  **Aesthetic Intent**: Explain the "why" behind the design (e.g., why a specific animation is used to create an "alive" feel).
